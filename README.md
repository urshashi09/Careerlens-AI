# Careerlens AI

Careerlens AI is a full-stack AI interview preparation platform that generates personalized interview reports from a candidate's resume, self-description, and target job description. It helps users understand their role fit, practice tailored interview questions, identify skill gaps, and follow a structured preparation roadmap.

## Highlights

- Secure authentication with JWT and cookie-based sessions
- PDF resume upload and parsing
- AI-powered report generation using Google's Gemini API
- Tailored technical and behavioral interview questions
- Model answers and interviewer intentions for each question
- Match score with score-aware UI
- Skill gap analysis with severity levels
- 7-day personalized preparation roadmap
- Report history page for previously generated reports
- Responsive glassmorphism UI built with React and SCSS

## AI Model Strategy

The report generation service uses:

- Primary model: `gemini-2.5-flash`
- Fallback model: `gemini-2.5-flash-lite`

If the primary model returns a temporary high-demand or `503 UNAVAILABLE` error, the backend retries the same structured request with the fallback model.

## Local Setup

### Prerequisites

- Node.js
- MongoDB local instance or MongoDB Atlas URI
- Google Gemini API key

### Backend

```bash
cd backend
npm install
```

Create a `.env` file in the `backend` directory:

```env
MONGO_URI=your_mongodb_connection_string
JWT_SECRET=your_jwt_secret
GEMINI_API_KEY=your_gemini_api_key
```

Start the backend server:

```bash
node server.js
```

The backend runs on:

```text
http://localhost:3000
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

The frontend runs on:

```text
http://localhost:5173
```

## API Routes

### Auth

| Method | Route | Description |
| --- | --- | --- |
| `POST` | `/api/auth/register` | Register a new user |
| `POST` | `/api/auth/login` | Log in a user |
| `GET` | `/api/auth/logout` | Log out the current user |
| `GET` | `/api/auth/getme` | Get the authenticated user |

### Reports

| Method | Route | Description |
| --- | --- | --- |
| `POST` | `/api/user/interview` | Generate a new interview report from resume, self-description, and job description |
| `GET` | `/api/user/interview/:id` | Fetch a single report by ID |
| `GET` | `/api/user/allReport` | Fetch all reports generated by the authenticated user |

The report generation endpoint expects `multipart/form-data` with:

- `resume`: PDF file
- `jobDescription`: target job description
- `selfDescription`: candidate self-description

## Key Implementation Details

- Authentication state is managed with React context on the frontend and JWT cookies on the backend.
- Resume files are uploaded with Multer and parsed with `pdf-parse`.
- Gemini responses are requested as structured JSON using a response schema.
- Skill gap severities are normalized before saving to MongoDB to prevent invalid enum values.
- Report history is scoped to the authenticated user.
